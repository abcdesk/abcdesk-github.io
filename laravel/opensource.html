    count(): Parameter must be an array or an object that implements Countable

    分析：查询官方手册发现7.2版本的php有更新，count传入非数组的变量会出现warning；本代码中query->wheres 为 NULL，count(NULL)则报错。

    解决方案:

    定位到出错文件/vendor/laravel/framework/src/Illuminate/Database/Eloquent/Builder.php

        // 1231行 $originalWhereCount = count($query->wheres);修改如下：
    if ($query->wheres != null) {
        $originalWhereCount = count($query->wheres);
    } else {
        $originalWhereCount = 0;
    }

    count(): Parameter must be an array

    分析：count函数在PHP7.2版本中已经是严格要求传入的参数类型了，只能是数组或者 Countable 对象。

    解决方案:

    1.降低php版本至于7.2以下

    2.修改代码:

    # 保证至少有一个条件
$query->where('deleted_at','==','null');
# 在执行前先判断
if(is_array($arr)){
  count($arr);
}
# is_null()判断是否为空
if ((is_null($constraints)) ? 0 : sizeof($constraints)) {
}
# 如果统计的是字符串，可替换为strlen()函数
$arg = substr($arg,0,count($arg)-2);
修改为：
$arg = substr($arg,0,strlen($arg)-1);
# 前面加(array)转换为数组
return count((array) $query->wheres) > $originalWhereCount;

Laravel QA问题及解决方案

composer install一直报错，无法完成

分析：大部分是因为composer被墙造成

解决方案:

  1. 设置国内镜像 composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/【详情】

    2. 注：若已设置，则略过。

  2. 修改项目根目录composer.json文件，如果已设置composer源，则删除；
  3. 删除项目根目录composer.lock文件，重新运行composer install即可。
