Git Pull获取远程分支

远程仓库的默认名称是origin

$ git pull <远程主机> <远程分支>:<本地分支>
$ git pull origin master:my_test #将origin仓库的master分支拉取并合并到本地的my_test分支上
$ git pull origin master #如果省略本地分支，则将自动合并到当前所在分支上

****git 从远程仓库获取所有分支***

方法一：

$ git clone https://github.com/laravel/laravel.git
$ cd project
$ git branch -a  #列出所有分支名称如下：

* master remotes/origin/3.0 remotes/origin/5.0 remotes/origin/5.1 remotes/origin/5.2 remotes/origin/5.3 remotes/origin/5.4 remotes/origin/5.5 remotes/origin/5.6 remotes/origin/5.7 remotes/origin/5.8 remotes/origin/HEAD -> origin/master remotes/origin/develop remotes/origin/master 

$ git checkout -b 5.1 origin/5.1

作用是checkout远程的dev分支，在本地起名为dev分支，并切换到本地的dev分支

$ git checkout -b release origin/release
$ git checkout dev  #切换回dev分支，并开始开发。

方法二：

$ git branch -r | grep -v '\->' | while read remote; do git branch --track "${remote#origin/}" "$remote"; done
$ git fetch --all
$ git pull --all

$ git pull origin 5.5:5.5 #将origin仓库的5.5分支拉取并合并到本地的5.5分支上
$ git checkout -b dev origin/daily/1.4.1

$ git remote add origin git@server-name:path/repo-name.git #关联远程库
$ git push -u origin master #第一次推送master分支的所有内容
$ git push origin master    #推送最新修改

多人协作的工作模式通常是这样： 首先，可以试图用git push origin <branch-name>推送自己的修改； 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并； 如果合并有冲突，则解决冲突，并在本地提交； 没有冲突或者解决掉冲突后，再用git push origin <branch-name>推送就能成功！
